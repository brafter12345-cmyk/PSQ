For security teams investigating traffic from this scanner. This page is intentionally public, indexable, and stable — bookmark or share it with WAF / SIEM teams who encounter requests with the User-Agent Phishield-Scanner/1.0.

What this scanner does

The Phishield Scanner performs passive, read-only external assessment of publicly accessible infrastructure for the purpose of cyber insurance underwriting and risk reporting. It is operated by Phishield UMA (Pty) Ltd, an Authorised Financial Services Provider (FSP 46418), as part of a managed cyber cover product.

  • Only publicly accessible endpoints are probed (no authenticated traffic, no credentials submitted)
  • No exploit attempts; no payload submission; no fuzzing
  • HTTP traffic is HEAD-first for path-existence checks; GET only when content is required for analysis
  • No POST requests against the scanned target (POST is reserved for our own API ingress on this service)
  • Every request announces itself via the User-Agent below

What the scanner accesses

  • DNS records: A, AAAA, MX, TXT, NS, CAA, SPF, DMARC, DKIM (read-only)
  • TLS / SSL certificate chain via sslyze (no handshake mutation)
  • HTTP response headers and homepage HTML (read-only)
  • Probes for standard well-known paths: /.well-known/security.txt, /robots.txt, /sitemap.xml, /privacy, etc.
  • Probes for sensitive-file exposure patterns: /.env, /.git/config, /wp-config.php.bak, etc. — used to grade security hygiene, not to exfiltrate
  • Subdomain enumeration via certificate transparency logs (no traffic to the target)
  • External data sources (Shodan, HIBP, VirusTotal, etc.) — these calls do NOT touch the target

WAF / Bot-Manager handling

If the scanner detects WAF / bot-manager intervention (challenge pages, 403 / 429 / 503 response patterns, consistent timeouts), the affected sections of the report are explicitly flagged as partial coverage. The scanner does NOT attempt to evade WAF rules.

How to verify a request is from this scanner

  1. Check the request's User-Agent header — legitimate scanner traffic always includes Phishield-Scanner/1.0 (+https://phishield.com/scanner-info).
  2. Visit the URL embedded in the User-Agent (this page). If you reached it, the scanner identity is verified.
  3. Check that the source IP is within Render.com's public egress range. (Render documents these at render.com/docs/static-outbound-ip-addresses.)
  4. For doubt or escalation, contact the email below — we will confirm whether a specific scan timestamp corresponds to a paid Phishield assessment of your organisation.

Allowing the scanner past your WAF

This is optional. The scanner is designed to operate gracefully under WAF intervention — affected sections are flagged in the report, not faked. However, if your security team chooses to verify and allow the scanner explicitly, the safest rule is:

Allow requests where:
  User-Agent matches "^Phishield-Scanner/" AND
  Source IP is within Render.com's documented egress range
Rate limit:
  Cap at 5 requests / second / scanner identity

Do NOT whitelist by User-Agent alone — User-Agents are forgeable. Always pair with source IP.

Stopping the scanner

If your organisation does not consent to being scanned, contact the email below with the affected domain. The scanner respects opt-out requests from the registered owner of the target domain within 24 hours.
